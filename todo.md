# 燕中校友数字母港：未来拓展方向

本文档只记录尚未实现、值得继续评估的方向。事项按优先级排列，不代表必须全部开发；进入实施前应补充需求、数据边界和验收标准。

## P0 - 上线运营闭环

- [ ] 为管理员增加待审核账号数量提醒和快速审核入口，减少新用户等待时间。
- [ ] 为身份审核补充可填写的驳回原因，并允许用户修改资料后重新提交。
- [ ] 评估审核结果邮件通知；发送失败不得影响审核事务本身。
- [ ] 在生产环境演练数据库、上传目录和应用版本的完整恢复流程，并记录恢复耗时。
- [ ] 建立每次发布后的登录、注册、邮箱验证、审核和管理员操作检查清单。

## P1 - 星空彩蛋正式版

- [ ] 让人物、火箭和会徽共享同一个带 `starId` 的 360 星数据源，实现真正的星点守恒。
- [ ] 固定 12 颗强调星在掌心、焊点、火箭核心和会徽高光之间的接力轨迹。
- [ ] 获得授权后，将校友团队设计的品牌火箭转换为正式点阵。
- [ ] 在画面时间轴稳定后制作约 52 秒原创配乐与星光、焊接、点火、升空音效。
- [ ] 增加静音控制、音频预加载、加载失败降级和用户本地偏好记忆。
- [ ] 在更多真实手机上验证连续动作、发热、帧率和低性能降级效果。

详细方案见 `docs/starfield-contribution.md`。

## P1 - 账号与校友服务

- [ ] 增加用户可查看的身份申请详情、提交时间和历史审核结果。
- [ ] 允许认证用户在隐私设置中决定联系方式是否对其他认证校友可见。
- [ ] 完善旧资料认领、重复账号合并和合并后的审计与通知流程。
- [ ] 评估电子校友证正式签发规则、唯一编号、二维码用途和防伪校验方式。
- [ ] 为活动增加名额候补、现场口令签到和报名状态通知。

## P1 - 小程序与跨端底座

- [ ] 完成微信小程序真实 AppID、登录凭据和生产域名配置。
- [ ] 补齐小程序个人资料、身份认证、新闻、活动、报名与取消的完整 UI 流程。
- [ ] 统一网站与小程序的账号状态、错误码、分页和缓存契约。
- [ ] 建立小程序真机验收矩阵，覆盖微信登录、弱网、HarmonyOS、iOS 和 Android。
- [ ] 评估未来 App 或其他端复用现有 API 的成本，不提前引入无实际需求的跨端框架。

## P2 - 内容与社区能力

- [ ] 建立校友新闻、活动、故事和成就内容的编辑规范与发布流程。
- [ ] 评估按城市、届别和行业进行校友发现的隐私边界与最小展示字段。
- [ ] 设计非盈利、学校支持场景下的校友互助与资源对接规则，暂不引入收费功能。
- [ ] 为学校、校友会和共建者建立可复用的内容运营数据看板。

## P2 - 工程与运维

- [ ] 为网页登录状态、邮箱验证和管理员审核增加稳定的端到端 CI 测试环境。
- [ ] 为星空演出增加可跳转到指定时间点的视觉回归测试，避免每次等待完整播放。
- [ ] 增加服务错误率、邮件发送失败、登录限流和数据库备份失败告警。
- [ ] 数据量或并发接近 SQLite 运维边界时，再评估迁移到 PostgreSQL；迁移前先完成容量测量。
- [ ] 定期升级 Next.js、Prisma、React、Sharp 等核心依赖，并在隔离环境验证 migration 与生产构建。

## 暂不计划

- [ ] 不引入收费、捐赠、招聘交易或其他盈利功能。
- [ ] 不建立复杂的地方分会、多级组织和细粒度角色体系。
- [ ] 不为了技术统一而重写当前稳定的网站底座。
- [ ] 不在缺少授权时上传品牌火箭原图、校友个人资料或学校内部数据。
