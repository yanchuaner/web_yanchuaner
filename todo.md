# 燕中校友数字母港 - 共建优化待办路线图

本文档基于共建者提出的 UI/UX、全栈安全、性能与工程化三大方向，结合当前项目现状整理而成。目标不是追逐功能堆叠，而是在“高频访问的组织门户”和“包含校友信息交互”的双重场景下，持续提升访问体验、隐私安全和维护效率。

## 当前基线

项目已具备的基础能力：

- Next.js 15 App Router + TypeScript + Tailwind 语义设计令牌。
- Prisma 7 + better-sqlite3 + SQLite WAL，适合公益站点低运维部署。
- HMAC-SHA256 httpOnly cookie 鉴权，密码使用 bcrypt。
- 管理员后台、内容审核、校友名册、故事投稿、校友纠错等核心闭环。
- `readJsonBody` 请求体大小限制、三层限流、CSV 导入防御、图片 Sharp 重编码。
- Open Graph/metadata、standalone 构建、Nginx/systemd 部署文档。

需要继续补齐的重点：

- 移动端首屏和动态列表的等待体验。
- 安全响应头、CSRF 策略、敏感字段加密/脱敏边界。
- 图片资源/CDN/缓存策略。
- GitHub Actions、自动化测试和部署前检查。
- 近期“届别/班级纯数字存储”改动后的存量数据清洗与运营闭环。

## P0 - 近期必须处理

### 1. 存量身份字段清洗

背景：PR #23 已统一届别、班级存储口径：数据库只存 `2025`、`1`，展示层格式化为 `2025届`、`1班`。

- [x] 将 `npm run normalize-identity-fields` 改为生产部署包可执行的 Node.js 脚本，并支持 `-- --dry-run`。
- [ ] 生产部署前备份生产 SQLite 数据库和上传目录。
- [ ] 在生产环境先执行 dry-run，再执行 `node scripts/normalize_identity_fields.js`。
- [ ] 抽查 `User`、`WhitelistRoster`、`Achievement`、`AlumniCorrectionRequest` 中的旧值是否完成清洗。
- [ ] 验证注册、后台新增/编辑校友、校友纠错审核、成果管理均能保存纯数字格式。

验收标准：

- 数据库中不再新增 `2025届`、`1班` 这类带后缀值。
- 前台仍正常展示为 `2025届`、`1班`。
- 历史待审纠错申请通过后写回纯数字。

### 2. 修复 Prisma 7 seed 配置漂移

背景：当前 `package.json` 中有 `prisma.seed`，但 Prisma 7 读取 `prisma.config.ts` 后，`prisma db seed` 在 build 中提示 `No seed command configured`，导致 `npm run build` 的 seed 步骤实际跳过。

- [x] 在 `prisma.config.ts` 中配置 seed 命令。
- [x] 确认 `npm run build` 会实际执行 `prisma/seed.ts`（已在 WSL 原生临时目录用临时 SQLite 验证）。
- [x] 继续保留 `npm run seed` 作为直接播种入口。

验收标准：

- WSL/Linux 下 `npm run build` 输出真实 seed 日志。
- 多次 build 不产生重复白名单或故事数据。

### 3. 生产安全响应头补强

当前已有 `X-Content-Type-Options`、`X-Frame-Options`、`Referrer-Policy`。继续补齐：

- [x] 在 Next headers 中启用 HSTS：`Strict-Transport-Security`。
- [x] 设计并灰度启用 CSP，先使用 `Content-Security-Policy-Report-Only` 收集误伤。
- [x] 明确图片、字体、上传图片、Resend/接口请求等允许来源。
- [x] 更新 `docs/security.md`、`docs/deployment.md` 和 `docs/operations-guide.md`。

验收标准：

- HTTPS 站点返回 HSTS。
- CSP 不阻断现有页面、地图、上传图片、后台操作。
- 安全扫描不再提示基础安全头缺失。

### 4. 上线前调试接口与上传面收口

- [x] 删除无前端引用的公开测试邮件接口 `/api/test-email`，避免生产环境被滥用发信。
- [x] 删除重复的校友证个人背景上传接口 `/api/alumni/certificate/upload-bg`。
- [x] 校友证个人背景改为浏览器本地读取和生成，不再把个人自定义背景写入服务器。
- [x] 保留管理员专用 `/api/settings/card-bg/upload` 作为站点默认纪念卡背景配置入口。

验收标准：

- 认证校友生成纪念卡时不会产生服务器端上传文件。
- 生产路由清单中不再出现公开调试发信接口。

### 6. 骨架屏与加载状态统一

- [x] 为动态列表页建立统一 Skeleton 组件或 CSS 组件类。

### 8. 深色模式策略评估

当前 V2.0 已是“星空紫深色玻璃拟态”主视觉，不建议短期再做完整亮/暗双主题。

- [ ] 先梳理仍残留的浅色硬编码类名，替换为语义令牌。
- [ ] 评估是否需要“高对比/护眼模式”，而不是完整 Dark Mode 切换。
- [ ] 若实施主题切换，必须基于 `globals.css :root` 和 `tailwind.config.ts` 的语义令牌，不在页面级硬写色值。

验收标准：

- 新代码不新增裸写十六进制色值。
- 深色背景下所有正文、表单、弹窗对比度可读。

## P1 - 全栈安全与隐私保护

### 9. CSRF 策略审计

当前 cookie 使用 httpOnly + sameSite，但敏感 POST/PATCH/DELETE 仍应做一次专项审计。

- [x] 列出所有敏感写接口：登录、登出、注册、改密码、资料编辑、活动报名、投稿、后台 CRUD。
- [x] 确认 sameSite、Origin/Referer 校验、CSRF token 三种方案的适用边界。
- [x] 优先给后台和账号安全操作增加 Origin/Referer 校验或 CSRF token。
- [x] 编写最小回归测试或 smoke 脚本覆盖失败路径。

验收标准：

- 跨站表单无法触发敏感写操作。
- 不影响正常微信/移动浏览器访问。

### 10. 敏感信息存储与展示分级

- [x] 梳理手机号、邮箱、联系方式、证书编号、身份标识等字段的存储与返回路径。
- [x] 对公开 API 和高敏管理员列表继续执行显式 `select` 脱敏策略。
- [ ] 评估联系方式是否需要应用层加密存储，至少做到后台可见、前台最小展示。
- [ ] 后台导出 CSV 时继续保留公式注入防护，并增加导出权限说明。

验收标准：

- 未登录或未验证访问不返回联系方式。
- 普通校友只能看到必要字段，管理员才可看完整联系方式。

### 11. XSS 与富文本安全

- [x] 审查新闻、活动、故事、投稿内容的渲染方式，当前均使用 React 文本渲染与 `whitespace-pre-wrap`，不执行 HTML。
- [x] 确认不存在 `dangerouslySetInnerHTML`；后台正文输入改为“支持换行”文案，避免误导为富文本。
- [x] 对用户输入的标题、简介、留言统一长度限制和展示转义；封面和记忆馆图片路径限制为站内上传路径。
- [x] 上传图片继续走 Sharp 重编码，禁止直接信任原始文件；服务端统一限制 JPEG/PNG/WebP 并输出重编码 JPEG。

验收标准：

- 用户提交 `<script>`、事件属性、危险链接不会执行。
- 富文本内容仍能保留必要排版。

### 12. 限流、审计与告警

项目已有限流和审计基础，下一步是运营可见性。

- [ ] 给登录、注册、验证邮件、投稿、纠错、活动报名补齐限流表。
- [x] 对管理员关键操作继续写入 `AuditLog`，补充缺失接口（含用户邮件/密码操作、信息修正审核）。

## 每次改动后的验证清单

代码改动默认执行：

```bash
npx prisma generate
npx tsc --noEmit
npm run lint
```

涉及构建、路由、Next.js 动态渲染、Prisma 或全局组件时，在 WSL/Linux 执行：

```bash
npm run build
```

涉及数据清洗、seed 或部署流程时额外执行：

```bash
npm run seed
npm run normalize-identity-fields
```

如果验证受环境限制无法执行，必须在 PR 或交付说明中写明原因和替代检查。
