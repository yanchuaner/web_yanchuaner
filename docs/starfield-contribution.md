# 星空彩蛋演出设计书

最后更新：2026-07-11

设计阶段：V2 第二次完善进行中

当前线上版本：约 33.5 秒初版

## 1. 项目定位

`/starfield` 是燕中校友数字母港中的隐藏互动演出，灵感来自深圳市燕川中学的航天特色与无人机编队表演。它不是营销页，不进入常规导航；用户在首页连续点击三次站点标识后进入。

演出讲述一条完整故事线：

> 一位少年从仰望星空开始，接住一束微光，亲手建造火箭，将自己化作星流送它升空。离散的星光最终再次相聚，组成燕中校友会会徽。

核心隐喻是“每位校友都是同一片星空中的一颗星”。人物、火箭与会徽不是三套互不相关的图形，而是同一群星在不同人生阶段中的重新编队。

## 2. 设计目标

V2 目标不是简单延长初版，而是把“定格画面切换”升级为一段约 52 秒的连续星空演出：

- 人物动作具有准备、发力、停顿和重心变化，不再从站立直接切到俯身。
- 火箭按部件逐步成型，不再一次出现完整轮廓。
- 主角星群全程保持身份与数量，人物退场时自然流入火箭，火箭升空后再组成会徽。
- 文案参与叙事，但不成为持续显示的进度提示。
- 视觉时间轴确定后再制作配乐，避免反复剪辑音频。
- 保持移动端单屏、可跳过、可降级，不以牺牲可用性换取效果。

## 4. 星群系统

### 4.1 数量

正式演出采用以下星点预算：

| 星群 | 数量 | 用途 |
| --- | ---: | --- |
| 主角星群 | 360 | 人物、火箭、星流与最终会徽 |
| 强调星 | 12 | 主角星群的子集，用于掌心、焊点、核心与会徽高光 |
| 环境星群 | 桌面 144 / 移动端 88 | 远景闪烁和轻微视差，不参与主叙事 |

360 对应完整圆周，也能为人物关节、火箭细节和会徽轮廓提供足够采样点。12 颗强调星包含在 360 颗主角星中，不额外增加节点。

### 4.2 星点守恒

每颗主角星拥有固定 `starId`。演出过程中只改变位置、亮度、大小和色温，不随场景随意销毁或新建：

```text
散落星光
  → 人物轮廓
  → 掌心与建造星流
  → 人物和火箭
  → 火箭核心与尾迹
  → 会徽轮廓、轨道和中心图案
```

环境星不受守恒规则约束。粒子火花、冲击波和尾焰可以是短生命周期特效，但不能承担主体轮廓。

### 4.3 星点身份

主角星按职责分组，但在不同场景中允许重新分配目标：

- `core`：12 颗强调星，承担关键视觉接力。
- `body`：人物躯干与火箭箭体的主要结构。
- `limb`：人物四肢，后续优先进入火箭轮廓和尾翼。
- `detail`：面部朝向、手部、舷窗、焊点和会徽中心细节。
- `flow`：用于填补密度并形成星流过渡。

相邻画面优先保持局部对应关系，避免大量星点交叉穿越。需要重新分配时使用最近邻匹配，并为核心星保留人工指定轨迹。

## 5. V2 分镜与文案

目标总时长为 52 秒，允许实现阶段在不改变故事顺序的前提下浮动约 2 秒。

| 时间 | 场景 | 动作设计 | 屏幕文案 |
| --- | --- | --- | --- |
| 0～5.5 秒 | 星光成形 | 环境星逐渐苏醒，主角星从四周聚成站立少年 | 最初，只是仰望。 |
| 5.5～10 秒 | 苏醒仰望 | 头部先抬起，肩部和胸腔随后展开，视线追随核心星 | 无 |
| 10～14 秒 | 接住微光 | 核心星下降；少年转头、抬臂、伸手，星停在掌心 | 后来，我们试着接住一束光。 |
| 14～18 秒 | 转身准备 | 头部先看向空地，肩、髋、脚依次转向，掌心星拉出短轨迹 | 无 |
| 18～23 秒 | 缓慢俯身 | 膝盖弯曲、重心下沉、躯干前倾，手臂最后伸向工作区 | 把遥远的想象，亲手变成方向。 |
| 23～31 秒 | 建造火箭 | 发动机、箭体、舷窗、尾翼依次编队；少年手臂往返焊接 | 无 |
| 31～35 秒 | 起身检视 | 少年收回手臂、缓慢起身、后退半步，火箭完成一次呼吸点亮 | 有些梦想，从来不是一个人的远方。 |
| 35～39 秒 | 星光相承 | 人物从手部、胸口到全身化为星流，汇入火箭核心与外壳 | 我们把自己，也写进了航程。 |
| 39～42 秒 | 点火蓄力 | 先静止，再出现低频震动、能量环、地面光晕和尾焰 | 无 |
| 42～45 秒 | 火箭升空 | 冲击波展开，火箭加速离场，镜头产生轻微下沉和速度感 | 无 |
| 45～49 秒 | 群星归位 | 火箭释放主角星，依次组成会徽外环、内环、轨道和中心图案 | 散作群星，仍在同一片天空相逢。 |
| 49～52 秒 | 最终落版 | 会徽整体呼吸一次，背景安静下来，最终寄语出现 | 愿你奔赴群星，也永远记得从燕川出发。 |

文案规则：

- 同一时刻最多一句，正文最多两行。
- 文案直接置于星空中，不使用卡片、字幕条或明显边框。
- 通过稀疏星尘聚合出现，离场时重新散开，不做普通页面式切换。
- 点火、升空和火箭主要建造阶段保持无文字，让动作成为焦点。
- 阶段名称只保留给无障碍播报与开发调试，不作为正式演出常驻 UI。

## 6. 人物连续动作

人物动作由一组有明确过渡关系的姿态组成：

```text
gather
→ stand
→ lookUp
→ reachStar
→ catchStar
→ turn
→ bendStart
→ bendDeep
→ weldReach
→ weldSweep
→ standUp
→ stepBack
→ dissolve
```

每个姿态继续由头、颈、肩、肘、腕、髋、膝、踝等关节生成，不复制整段 SVG。

动作必须体现局部先后关系：

- 抬头：眼前星光先移动，头部跟随，肩胸最后展开。
- 转身：头先转，肩随后，髋与脚完成重心调整。
- 俯身：膝与髋先下降，躯干再前倾，手臂最后伸出。
- 起身：手臂先收回，躯干抬起，腿部伸直，最后后退观察。
- 消散：从接触火箭的手开始，经胸口向全身扩散，避免人物整体同时淡出。

姿态间不仅改变坐标，也应设置不同缓动。准备动作偏慢，伸手和焊接带轻微加速，起身与观察需要留出呼吸停顿。

## 7. 火箭建造与正式素材

通用火箭继续作为 V2 的制作基线，建造顺序固定为：

```text
发动机与中轴
→ 主箭体
→ 舷窗与核心
→ 尾翼
→ 表面能量线路
→ 完成点亮
```

取得学弟团队设计的正式火箭图后：

1. 确认原作者允许公益网站展示、点阵化处理和开源仓库收录。
2. 优先获取 SVG、AI、PSD 或透明 PNG；JPG 需先校正透视并抠图。
3. 提取外轮廓、舷窗、标志、尾翼、喷口与关键装饰。
4. 将轮廓重采样为 360 星预算内的目标点，不直接叠加实体图片。
5. 保留现有故事、人物动作、点火、升空和会徽阶段，只替换火箭 formation。

## 8. 会徽高潮

会徽不显示中文或英文，只保留外轮廓、轨道和中心图案。360 颗主角星按以下顺序归位：

1. 外环从火箭尾迹两侧合拢。
2. 内环与两侧节点落位。
3. 五段轨道按同一方向依次连接。
4. 中心图案最后成型。
5. 12 颗强调星依次闪亮，随后全体同步呼吸一次。

连线只作为星点之间的微弱能量轨迹，亮度不得压过星点主体。最终会徽应在手机上保持清晰、近似圆形并完整居中。

## 9. 交互与音频预留

- 首次观看不显示跳过；完整观看一次后允许跳过。
- 发射次数只保存在 `localStorage`，不写入数据库。
- 返回、跳过、静音和重播按钮避开移动端安全区域。
- 用户点击发射是未来启动音频的唯一入口，满足浏览器自动播放限制。
- 音乐在 V2 画面和时间轴锁定后制作，目标长度约 52 秒。
- 音频层预留背景音乐、星光、焊接、点火、升空五条逻辑轨道；加载失败不影响画面。
- 静音状态保存在本地，默认尊重用户选择。

## 10. 技术实现边界

当前技术继续采用 React、TypeScript、SVG 星点与 CSS 动画，不为 V2 引入大型动画或 3D 引擎。

| 文件 | 当前职责 |
| --- | --- |
| `src/components/starfield/StarfieldExperience.tsx` | 总状态机、时间轴、跳过、重播和本地次数 |
| `src/components/starfield/RocketWorkshop.tsx` | 人物姿态、通用火箭与星流目标点 |
| `src/components/starfield/RocketWorkshop.module.css` | 星点移动、焊接、点火与升空效果 |
| `src/components/starfield/ConstellationEmblem.tsx` | 无文字会徽目标点和连线 |
| `src/components/starfield/ConstellationEmblem.module.css` | 会徽汇聚、描边和点亮 |

V2 实现要求：

- 总时间节点仍由 `StarfieldExperience` 统一维护。
- 子组件不得创建相互独立的长时间轴。
- 场景切换由离散状态控制，场景内部动作使用连续插值或连续关键姿态。
- 主角星的 `starId`、分组与强调身份贯穿工作坊和会徽阶段。
- 不通过无限增加 DOM 节点弥补图形设计不足。
- `prefers-reduced-motion` 开启时快速进入终幕，不强制播放 52 秒动画。

## 12. 验收标准

视觉验收：

1. 不看文案也能理解“仰望、接星、俯身建造、起身、点火、升空、重组”七个关键动作。
2. 人物俯身和起身是连续动作，不是两个定格轮廓直接切换。
3. 主角星在人物、火箭和会徽之间没有明显整批消失或凭空生成。
4. 会徽无文字，轮廓与中心图案在手机上清晰可辨。
5. 375×667、390×844 和桌面视口均为单屏，无控件遮挡或页面溢出。

技术验收：

1. 首次完整播放、再次播放跳过、重播均可用。
2. 页面不可被搜索引擎索引，隐藏入口行为不变。
3. `prefers-reduced-motion` 能快速完成。
4. 移动端目标为流畅播放；低性能设备允许减少环境星和滤镜，不减少主体星。
5. 工作坊退场后停止无用的持续动画。
6. 会徽 SVG 不包含 `<text>` 节点。

提交前运行：

```bash
npx tsc --noEmit
npm run lint
npm run audit:ui-tokens
```
