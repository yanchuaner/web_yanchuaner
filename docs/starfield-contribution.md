# 星空彩蛋专项共建指南

最后更新：2026-07-11

## 项目定位

`/starfield` 是燕中校友数字母港中的隐藏互动演出，灵感来自深圳市燕川中学的航天特色和无人机编队表演。它不是独立营销页，也不进入常规导航；首页站点标识连续点击三次可进入。

当前版本为可体验、可继续共建的初版。核心故事是：一位由星光组成的少年仰望星空、接住微光、亲手绘制并点亮火箭，最终化为星流送火箭升空；升空后的星群重新组成校友会会徽。

## 当前分镜

完整演出约 33.5 秒，首次观看完整播放，完成一次后允许跳过。

| 时间 | 状态 | 画面 |
| --- | --- | --- |
| 0～3.0 秒 | `personGather` | 散落星光组成少年轮廓 |
| 3.0～6.2 秒 | `personAwake` | 少年抬头并缓慢举手 |
| 6.2～9.4 秒 | `starCatch` | 核心星落入掌心 |
| 9.4～13.2 秒 | `rocketTrace` | 掌心星流绘出火箭轮廓 |
| 13.2～17.4 秒 | `welding` | 少年俯身焊接，火花沿结构亮起 |
| 17.4～20.4 秒 | `rocketReady` | 少年站起后退，检视完整火箭 |
| 20.4～22.6 秒 | `personDissolve` | 人物化为三股星流汇入火箭 |
| 22.6～24.4 秒 | `ignition` | 尾焰与能量环点亮 |
| 24.4～26.9 秒 | `launching` | 火箭升空，速度线出现 |
| 26.9～31.8 秒 | 会徽编队 | 星点汇聚、连线并点亮会徽 |
| 31.8～33.5 秒 | `reveal` / `complete` | 显示寄语、次数和重播入口 |

时间节点统一定义在 `src/components/starfield/StarfieldExperience.tsx`，不要在子组件中增加独立的长计时器。

## 技术结构

| 文件 | 职责 |
| --- | --- |
| `src/app/starfield/page.tsx` | 页面 metadata 与禁止索引配置 |
| `src/components/starfield/StarfieldExperience.tsx` | 总状态机、时间线、跳过、重播和本地发射次数 |
| `src/components/starfield/RocketWorkshop.tsx` | 340 颗星的目标点阵、人物姿态、火箭与星流生成 |
| `src/components/starfield/RocketWorkshop.module.css` | 点阵移动、呼吸、核心星、焊接、点火与升空动画 |
| `src/components/starfield/ConstellationEmblem.tsx` | 无文字校友会会徽星点编队 |
| `src/components/starfield/ConstellationEmblem.module.css` | 会徽汇聚、连线和点亮动画 |

`RocketWorkshop` 始终渲染同一组 340 颗星。每一幕只改变这些星的目标坐标、透明度、缩放与色调。相邻画面通过最近邻匹配重新排序，减少星点交叉和无意义的长距离飞行。

## 新增或修改点阵

点阵使用以下基础生成器组合：

- `linePoints`：直线点阵。
- `polylinePoints`：折线或肢体点阵。
- `ellipsePoints`：头部、光环、舷窗和能量环。
- `quadraticPoints`：星流和火箭曲线。
- `articulatedPerson`：根据头、肩、髋、手臂和腿部关节生成姿态。
- `rocketCorePoints`：通用火箭轮廓与内部特征。

新增姿态时应先在 `PERSON_POSES` 中定义关节坐标，再创建对应 formation。不要为一帧复制整段 SVG 标记。

所有 formation 最终必须经过 `normalizeFormation` 归一为 340 个目标点，并通过 `alignFormation` 与上一幕建立映射。

## 正式火箭素材接入

当前火箭是通用点阵。取得学弟团队设计的正式成品图后，建议按以下流程处理：

1. 确认原作者授权在公益网站及开源仓库中展示或二次处理。
2. 优先获取透明 PNG、SVG、AI 或 PSD；只有 JPG 时先完成抠图和透视校正。
3. 提取外轮廓、舷窗、标志、尾翼、喷口和关键装饰的采样点。
4. 将采样点转换为 `StarTarget[]`，替换 `rocketCorePoints` 的通用坐标。
5. 保持点数预算，避免通过无限增加 DOM 节点解决细节不足。
6. 分别检查人物绘制、完整成型、点火和高速升空阶段。

正式图不应直接作为实体图片叠在点阵上，否则会重新造成视觉语言割裂。

## 可认领任务

适合首次参与：

- 调整某一个人物姿态，使动作在手机上更容易辨认。
- 优化中文阶段文案或无障碍播报。
- 补充不同手机尺寸的截图和性能数据。
- 检查低亮度屏幕上的星点对比度。

适合熟悉前端动画的贡献者：

- 优化相邻 formation 的匹配算法，减少局部拥挤或交叉。
- 将正式火箭图转换为稳定、可维护的点阵数据。
- 优化低性能设备上的滤镜、粒子数量和帧率降级。
- 为演出增加可测试的状态时间线，不依赖真实等待 33.5 秒完成全部断言。

需要先讨论再实施：

- 用户主动开启的音效。
- 第二层隐藏互动或新彩蛋。
- 修改完整故事线、会徽结构或公开入口方式。
- 引入新的动画或物理引擎。

## 视觉与性能边界

- 人物、火箭和会徽统一使用星点编队，不加入大面积实体填充。
- 会徽内部不出现中文或英文文字。
- 页面保持单屏体验，不依赖上下滚动观看演出。
- 控件必须避开移动设备安全区域。
- 系统启用 `prefers-reduced-motion` 时直接进入寄语，不强制播放长动画。
- 工作坊退场后停止自身星点动画，避免与会徽阶段同时持续运行。
- 不新增裸写颜色；继续使用项目语义颜色变量。

## 本地验证

启动项目后访问 `http://localhost:3000/starfield`。至少检查：

1. 首次播放依次经过所有分镜，没有跳过按钮。
2. 每个动作在不看顶部文字时仍能被大致理解。
3. 重播时出现跳过按钮，并能直接进入完整终幕。
4. 终幕中工作坊已退场，会徽和寄语完整显示。
5. 375×667、390×844 和桌面视口没有横向或纵向溢出。
6. 会徽 SVG 中不存在 `<text>` 节点。
7. `prefers-reduced-motion` 模式能快速完成。
