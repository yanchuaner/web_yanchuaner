# 管理员使用手册

本手册覆盖后台所有功能模块的操作步骤，目标是让管理员能独立完成日常操作，遇到问题时知道先看哪里、如何回滚。

---

## 1. 登录后台

1. 打开浏览器访问 `https://yanchuaner.cn/admin/login`
2. 输入数据库中已创建的管理员用户名和密码
3. 登录成功后自动跳转到 `/admin` 控制面板，可看到各模块统计概览
4. 会话通过 httpOnly cookie (`yc_access_token`，role=admin) 维持，关闭浏览器后需重新登录

### 管理员账号管理

```bash
npm run create-admin
```

管理员账号存储在 `User` 表中，密码使用 bcrypt 哈希。登录会话由 `SESSION_SECRET` 签名；修改账号权限或密码后，用户需重新登录。

## 2. 新闻管理

路径：`/admin/news`

### 2.1 查看新闻列表

后台列表显示所有新闻（草稿 + 已发布），每条显示标题、状态、发布时间。

### 2.2 新建新闻

1. 点击"新建新闻"按钮 → 进入 `/admin/news/new`
2. 填写**标题**（必填）
3. 填写**摘要**（可选，显示在列表页卡片中）
4. 填写**正文内容**（必填，支持富文本）
5. 可选上传**封面图片**（支持 png / jpg / webp，建议尺寸 1200×630）
6. 点击"保存草稿"或"发布"
   - **保存草稿**：仅后台可见，前台不显示
   - **发布**：前台 `/news` 页面立即可见

### 2.3 编辑新闻

1. 在列表页找到目标新闻 → 点击"编辑" → 进入 `/admin/news/[id]`
2. 修改标题、摘要、正文或封面图片
3. 点击保存

### 2.4 发布 / 下架

在列表页切换状态开关：
- **发布** → 前台可见
- **下架（草稿）** → 前台不可见，后台保留

### 2.5 删除新闻

1. 在列表页点击删除按钮
2. 弹出二次确认框 → 确认后删除
3. **删除不可撤销**，建议先下架观察再决定是否删除

## 3. 活动管理

路径：`/admin/events`

### 3.1 新建活动

1. 点击"新建活动" → 进入 `/admin/events/new`
2. 填写**标题**（必填）
3. 填写**摘要**（可选）
4. 填写**正文内容**（必填）
5. 填写**地点**（可选，如"燕川中学体育馆"）
6. 选择**活动日期**（必填）和**结束日期**（可选）
7. 可选设置**最大参与人数**（达到上限后前台报名按钮自动禁用）
8. 可选上传**封面图片**
9. 点击"保存草稿"或"发布"

### 3.2 查看报名名单

1. 在活动列表找到目标活动 → 点击"报名名单" → 进入 `/admin/events/[id]/registrations`
2. 查看参与者姓名、联系方式、留言

### 3.3 导出报名名单（CSV）

1. 在报名名单页点击"导出 CSV"
2. 文件编码为 **UTF-8 BOM**（Excel 直接打开不乱码）
3. 以 `=`、`+`、`-`、`@` 开头的字段前自动加单引号，**防止 CSV 公式注入攻击**

### 3.4 删除活动

1. 在列表页点击删除按钮
2. **如果活动已有报名记录，系统会拒绝删除并返回提示**
3. 有真实报名数据的活动不建议删除，可改为下架状态

### 3.5 发布 / 下架

列表页切换状态开关，控制前台 `/events` 可见性。

## 4. 校友名单管理

路径：`/admin/alumni`

这是后台最核心的数据管理模块，涉及校友的姓名、届别、标签信息。

### 4.1 查看名单

- 列表分页显示所有已登记校友
- 支持按**姓名**和**班级**搜索

### 4.2 新增校友（单条）

1. 点击"新增"按钮
2. 填写**姓名**（必填）
3. 填写**届别**（如"2022级3班"）
4. 填写**标签**（格式：`大学名 | 专业 | 城市`，用 `|` 分隔，顺序固定）
   - 示例：`清华大学 | 计算机科学与技术 | 北京`
   - 标签中的城市信息用于校友地图城市聚合
5. 点击保存 → 自动同步到公开搜索 API 和地图 API

### 4.3 编辑校友

1. 点击行内"编辑"按钮
2. 修改姓名、届别或标签
3. 保存 → 自动同步到前台 API

### 4.4 删除校友

1. 点击行内"删除"按钮
2. 二次确认后删除
3. **删除不可撤销**

### 4.5 CSV 批量导入

这是新增大量校友数据的最快方式。

**文件要求**：
- 编码：**UTF-8**
- 格式：CSV（逗号分隔）
- 表头支持中文或英文：
  - 中文：`姓名,届别,标签`
  - 英文：`name,graduationClass,tags`
- 标签格式同单条新增：`大学名 | 专业 | 城市`

**导入步骤**：
1. 点击"导入"按钮 → 选择 CSV 文件
2. 系统自动解析并校验
3. **去重规则**：按 `name + graduationClass` 合并，同名学生同班级视为同一人
4. 导入完成后提示成功/失败数量

### 4.6 CSV 导出

1. 点击"导出"按钮 → 下载 CSV 文件
2. 文件名格式：`alumni-roster-YYYYMMDD.csv`
3. 编码：UTF-8 BOM
4. 防公式注入处理

## 5. 校友信息修改申请审核

路径：`/admin/alumni-corrections`

校友在前台 `/alumni/correction` 搜索自己姓名后，可提交信息修改申请。**申请不会直接修改数据库**，需管理员审核。

### 5.1 审核流程

1. 在列表页查看所有申请，支持按状态筛选：
   - **全部** / **待审核** / **已通过** / **已驳回**
2. 支持按姓名或联系方式搜索
3. 展开目标申请 → 查看**当前值**与**申请值**的对比：
   - 当前姓名 → 申请姓名
   - 当前届别 → 申请届别
   - 当前标签 → 申请标签
4. 选择操作：
   - **通过并应用**：自动更新 `WhitelistRoster` 中对应校友的信息
   - **驳回申请**：拒绝修改，数据保持不变
5. 可选填写**管理员备注**（记录审核原因）

### 5.2 审核注意事项

- 审核前先比对"当前值"和"申请值"是否合理
- 如有疑问，先驳回并留言说明原因，等待校友重新提交
- 审核后**不可撤销**

## 8. 校友成就墙管理

路径：`/admin/achievements`

管理校友成就页面（`/alumni/achievements`）的内容，支持分类筛选和发布控制。

### 8.1 新增成就

1. 填写**校友姓名**、**届别**（可选）
2. 填写**成就标题**（必填）
3. 选择**类别**（竞赛获奖、学术成果、职业成就、社会贡献、其他）
4. 填写**简介**、**机构/单位**、**年份**
5. 设置**排序号**（数字越小越靠前）
6. 点击保存

### 8.2 发布控制

成就默认状态为「草稿」，仅「已发布」状态会显示在前台页面。支持批量修改状态。

### 8.3 编辑 / 删除

同其他管理模块，支持完全 CRUD。

## 9. 燕中记忆管理

路径：`/admin/memories`

管理文化长廊展示的校园记忆展品（校园风景、毕业合影等），支持图文增删改、排序和图片上传。

**上传图片自动裁切为 16:9（2752×1548）**，与卡片展示比例一致。

### 9.1 新增展品

1. 填写**标题**（必填）
2. 填写**副标题**（可选）
3. 填写**描述**（可选）
4. 选择**图标**（六选一）：校门/教学楼/图书馆/操场/林荫道/相机
5. 上传或手动填写**图片路径**（可选）
6. 填写**图片 alt 说明**（可选）

### 9.2 排序

点击 ↑/↓ 箭头调整展品显示顺序。

### 9.3 编辑 / 删除

点击编辑按钮修改任意字段（支持部分更新），点击删除按钮移除展品（需确认）。

## 12. 每日操作建议

### 每天登录后先做什么

1. 查看 `/admin` 控制面板是否有异常统计
2. 检查待审核的修改申请（`/admin/alumni-corrections`）
3. 检查未处理的投稿（`/admin/posts`）
4. 查看活动报名是否有新增数据
5. 如有新闻或活动要更新，先保存草稿预览，确认无误后再发布

### 高风险操作前

- 确认当前有可用备份（参考 [BACKUP_GUIDE.md](BACKUP_GUIDE.md)）
- 批量导入前先在测试环境或小范围验证 CSV 格式
- 删除操作前确认数据是否已被引用

### 什么时候不要直接删除

- 活动已有报名记录时（系统会拒绝删除）
- 校友名单已被多处引用时
- 不确定数据是否已同步到前台时

## 13. 出现问题先看哪里

| 问题类型 | 排查方向 |
|----------|----------|
| 登录异常 | 检查账号状态、cookie 是否过期、`SESSION_SECRET` 是否正确、浏览器控制台错误 |
| 数据不一致 | 检查数据库和最近备份，确认是否有并发修改 |
| 上传失败 | 检查上传目录权限和磁盘空间 |
| 导入失败 | 检查 CSV 编码（必须是 UTF-8）、表头格式、文件是否损坏 |
| 页面空白或报错 | 查看浏览器控制台（F12），参考 [TROUBLESHOOTING.md](TROUBLESHOOTING.md) |

---

## 14. 误操作处理

### 如果不小心删除了重要数据

1. **立即停止操作**，不要再修改其他内容
2. **备份当前数据库**（防止二次损坏）
3. 从最近的备份恢复数据（参见 [BACKUP_GUIDE.md](BACKUP_GUIDE.md)）
4. 确认数据完整性后再恢复服务

### 原则

- 没有备份的情况下不要执行高风险操作（批量删除、导入覆盖等）
- 有真实用户报名数据的活动不删除
- 删除操作不可撤销，操作前务必确认
