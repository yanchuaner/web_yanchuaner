# 燕中数字母港 V2.0 管理员手册

本手册面向系统管理员，覆盖后台主要功能模块的操作步骤与安全规则。后台可能接触真实校友数据、活动报名信息和账号状态，所有导出、截图和人工沟通都应避免泄露个人隐私。

---

## 1. 登录后台

1. 浏览器访问 `https://yanchuaner.cn/login?redirect=/admin`
2. 输入管理员用户名和密码
3. 登录成功后自动跳转至 `/admin` 控制面板，可看到各模块统计概览
4. 会话通过 httpOnly cookie（`yc_access_token`，role=admin）维持，关闭浏览器后需重新登录

### 管理员账号管理

```bash
npm run create-admin
```

管理员账号存储在 `User` 表中，密码使用 bcrypt 哈希。登录会话由 `SESSION_SECRET` 签名。变更角色、账号状态或密码后，受影响用户需重新登录。

## 2. 控制面板

路径：`/admin`

登录后首先看到的概览页，展示后台各模块的关键统计数据（新闻数、活动数、待审核数等）。每日登录建议先浏览此页，确认无异常波动。

## 3. 故事审核工作台（V2.0 新增）

路径：`/admin/stories/pending`

这是 V2.0 最重要的新功能，取代了旧版的邮箱投稿流程。校友通过前台 `/alumni/stories` 页面或 `/me/submit` 个人中心提交故事后，故事进入 **PENDING（待审核）** 状态，管理员在此统一处理。

### 3.1 审核界面

- 每张卡片展示：故事标题、作者署名、标签（带 `#` 前缀）、正文预览、投稿日期
- 如果投稿人已登录且绑定了作者关系，卡片下方会显示**投稿人账户信息**：真实姓名、用户名、邮箱、届别、班级
- 卡片采用深色毛玻璃设计（玻璃拟态），与后台整体暗色主题一致
- 空状态时显示图标 + "暂无待审核的故事投稿"

### 3.2 审核操作

每个故事卡片底部有两个按钮：

| 按钮 | 操作 | 效果 |
|------|------|------|
| **通过** | 发送 `PATCH` 请求，`status: 'PUBLISHED'` | 故事立即出现在前台 `/alumni/stories` 页面 |
| **驳回** | 发送 `PATCH` 请求，`status: 'REJECTED'` | 故事不对外展示，投稿人在「我的投稿」中看到"已驳回"状态 |

- 点击通过或驳回后，该卡片从列表中移除（乐观 UI 更新）
- 操作成功显示绿色提示，失败显示红色提示
- **审核不可撤销**：通过或驳回后，状态变更立即生效，操作前请仔细阅读正文内容

### 3.3 审核背后的机制

审核操作与审计日志在同一数据库事务中写入，确保一致性：

1. 更新 `Story.status` 为 `PUBLISHED` 或 `REJECTED`
2. 同时插入一条 `AuditLog` 记录：
   - `action`：`story-review-published` 或 `story-review-rejected`
   - `targetType`：`Story`
   - `targetId`：故事 ID
   - `adminId`：操作管理员的用户 ID
   - `before`：审核前的状态快照（如 `{"status":"PENDING"}`）
   - `after`：审核后的状态快照（如 `{"status":"PUBLISHED"}`）

### 3.4 故事状态机总览

```
校友投稿 → PENDING → 管理员通过 → PUBLISHED（前台可见）
                   → 管理员驳回 → REJECTED（前台不可见，投稿人可见"已驳回"）

管理员直接创建 → 默认 PUBLISHED（无需审核）
投稿人撤销（PENDING/DRAFT 状态） → 删除投稿
```

## 4. 燕中故事管理（全量 CRUD）

路径：`/admin/stories`

管理所有故事记录（不限状态），支持标准增删改查。

### 4.1 新增故事

1. 点击"新建"按钮
2. 填写**标题**（必填，最长 120 字符）
3. 填写**作者**署名（可选，最长 50 字符）
4. 填写**标签**（逗号分隔，每个标签最长 30 字符，最多 10 个）
5. 选择**日期**
6. 填写**正文**（必填，最长 20000 字符）
7. 点击保存 → 管理员创建的故事默认为 PUBLISHED 状态，直接在前台可见

### 4.2 编辑 / 删除

点击编辑按钮修改任意字段，点击删除按钮移除故事（需确认，不可撤销）。

## 5. "我的投稿"（用户侧功能说明）

路径（用户端）：`/me/posts`

了解用户侧功能有助于管理员解答校友咨询。登录校友在个人中心可查看自己所有投稿：

| 状态 | 显示 | 用户可执行操作 |
|------|------|----------------|
| PENDING | "审核中"（琥珀色标签） | 可撤销投稿（删除） |
| PUBLISHED | "已发布"（翠绿色标签） | 不可操作 |
| REJECTED | "已驳回"（玫红色标签） | 不可操作 |
| DRAFT | "草稿"（灰色标签） | 可删除草稿 |

- 撤销投稿（PENDING）或删除草稿（DRAFT）会物理删除数据库记录
- 已发布和已驳回的投稿用户不可自行删除

## 6. 内容审核（统一审核面板）

路径：`/admin/posts`

跨类型的统一内容审核面板，覆盖 STORY（燕中故事）、EVENT（母港动态）、JOB（招聘信息）等多种内容类型。

- 操作按钮随状态变化：
  - PENDING → "发布" / "驳回"
  - PUBLISHED → "下架"
  - REJECTED → "重新发布"
  - 所有行均显示"删除"按钮

## 7. 新闻管理

路径：`/admin/news`

### 7.1 查看新闻列表

后台列表显示所有新闻（草稿 + 已发布），每条显示标题、状态、发布时间。

### 7.2 新建新闻

1. 点击"新建新闻"按钮 → 进入 `/admin/news/new`
2. 填写**标题**（必填）
3. 填写**摘要**（可选，显示在列表页卡片中）
4. 填写**正文内容**（必填，支持普通文本与换行；不要粘贴 HTML 或脚本）
5. 可选上传**封面图片**（支持 png / jpg / webp，建议尺寸 1200x630）
6. 点击"保存草稿"或"发布"
   - **保存草稿**：仅后台可见，前台不显示
   - **发布**：前台 `/news` 页面立即可见

### 7.3 编辑 / 发布与下架 / 删除

同标准 CRUD 操作。列表页切换状态开关控制前台可见性。删除不可撤销，建议先下架观察再决定。

## 8. 活动管理

路径：`/admin/events`

### 8.1 新建活动

1. 点击"新建活动" → 进入 `/admin/events/new`
2. 填写**标题**（必填）、**摘要**（可选）、**正文**（必填）
3. 填写**地点**（可选，如"燕川中学体育馆"）
4. 选择**活动日期**（必填）和**结束日期**（可选）
5. 可选设置**最大参与人数**（达到上限后前台报名按钮自动禁用）
6. 可选上传**封面图片**
7. 点击"保存草稿"或"发布"

### 8.2 查看与导出报名名单

1. 活动列表中找到目标活动 → 点击"报名名单" → 进入 `/admin/events/[id]/registrations`
2. 查看参与者姓名、联系方式、留言
3. 点击"导出 CSV"下载名单
   - 编码：UTF-8 BOM（Excel 直接打开不乱码）
   - 防公式注入：以 `=`、`+`、`-`、`@` 开头的字段前自动加单引号

### 8.3 删除活动

- 系统拒绝删除已有报名记录的活动（返回错误提示）
- 有真实报名数据的活动建议改为下架状态

## 9. 校友名单管理（含 CSV 批量导入 V2.0）

路径：`/admin/alumni`

这是后台最核心的数据管理模块。

### 9.1 查看名单

- 列表分页显示所有已登记校友（每页 50 条）
- 支持按**姓名**、**标签**、**班级**、**邮箱**搜索
- 支持按**届别**筛选

### 9.2 新增 / 编辑 / 删除（单条）

1. 点击"新增"按钮 → 弹窗表单
2. 必填：**姓名**（最长 50 字符）
3. 可选：**届别**（最长 50）、**班级**（最长 64）、**邮箱**（最长 254，需含 @）、**联系方式**（11 位手机号格式）、**标签**（最长 500 字符，格式：`大学名 | 专业 | 城市`）
4. 保存 → 自动同步到前台搜索 API 和地图 API
5. 删除前会检查是否有待处理的信息修改申请，如有则阻止删除

### 9.3 CSV 批量导入（V2.0 智能列检测）

这是批量新增校友数据的最快方式。

**文件要求**：
- 编码：**UTF-8**
- 格式：CSV（逗号分隔）
- 文件大小上限：**2MB**（防止 DoS 攻击）
- 至少包含表头行和一行数据

**V2.0 智能列检测**：
系统自动识别表头，支持中文和英文列名（不区分大小写，忽略空格/下划线/连字符）：

| 识别列名 | 映射字段 | 必填 |
|----------|----------|------|
| `姓名` / `name` | 姓名 | 是 |
| `届别` / `graduationClass` | 届别 | 否 |
| `班级` / `className` | 班级 | 否 |
| `邮箱` / `email` | 邮箱 | 否 |
| `标签` / `tags` | 标签 | 否 |
| `城市` / `所在城市` / `city` | 城市 | 否 |
| `院校` / `毕业院校` / `大学` / `university` | 院校 | 否 |
| `专业` / `就读专业` / `major` | 专业 | 否 |
| `行业` / `从事行业` / `industry` | 行业 | 否 |

**V2.0 城市名清理**：`城市` 列中的"市"后缀自动去除（如"深圳市"→"深圳"），用于地图聚合时城市名称标准化。

**导入步骤**：
1. 点击"导入"按钮 → 选择 CSV 文件
2. 系统自动解析表头、校验数据
3. **去重规则**：按 `name + graduationClass + className + email` 四字段组合去重
   - 匹配到已有记录 → 更新补充字段（标签、联系方式、城市、院校等），计入 skipped
   - 未匹配 → 创建新记录，计入 imported
4. 所有行在**单个数据库事务**中批量写入，减少 SQLite I/O 锁竞争
5. 导入完成后弹出结果通知：导入 X 条，跳过 Y 条，失败 Z 条（附错误详情，最多 20 条）

**校验规则**：
- 姓名为空的行自动跳过
- 姓名超过 50 字符的跳过并报错
- 邮箱不含 `@` 或超过 254 字符的跳过并报错
- 单个错误行不影响其他行的导入

### 9.4 CSV 导出

1. 点击"导出"按钮 → 下载 CSV 文件
2. 文件名格式：`alumni-roster-YYYYMMDD.csv`
3. 编码：UTF-8 BOM
4. 列：姓名、届别、班级、邮箱、联系方式、标签
5. 防公式注入处理

## 10. 校友信息修改申请审核

路径：`/admin/alumni-corrections`

校友在前台 `/alumni/correction` 搜索自己姓名后，可提交基础身份修正申请。**申请不会直接修改数据库**，需管理员审核通过后才会应用。

当前该入口仅处理姓名、届别、班级三项锁定字段；联系方式、城市、院校、专业、行业由校友在个人中心自行维护；邮箱变更暂不走自助修正申请，需联系管理员人工处理。

### 10.1 审核流程

1. 在列表页查看所有申请，支持按状态筛选：**全部** / **待审核** / **已通过** / **已驳回**
2. 支持按姓名或联系方式搜索
3. 展开目标申请 → 查看**当前值**与**申请值**的逐字段对比
4. 选择操作：
   - **通过并应用**：自动更新 `WhitelistRoster` 中对应校友信息
   - **驳回申请**：拒绝修改，数据保持不变
5. 可选填写**管理员备注**（记录审核原因）

### 10.2 审核注意事项

- 审核前仔细比对"当前值"和"申请值"是否合理，必要时通过申请中留下的联系方式二次确认
- 如有疑问，先驳回并留言说明原因
- 审核后不可撤销

## 11. 用户管理（V2.0 Root Admin 防护）

路径：`/admin/users`

管理已注册用户、审核校友认证、账号控制。所有敏感操作自动记录到 `AuditLog` 表。

### 11.1 用户列表

- 分页显示所有已注册用户
- 每个用户展示：用户名、邮箱、姓名、届别、班级、角色（GUEST/ALUMNI/ADMIN）、认证状态（PENDING/VERIFIED）、账号状态（ACTIVE/DISABLED）
- 支持按姓名、邮箱、用户名搜索

### 11.2 校友认证审核

用户注册后需管理员审核才能获得校友身份：

1. 找到状态为 PENDING 的用户
2. 点击操作按钮 → 选择"通过校友认证"
   - 系统将用户设为 ALUMNI 角色、VERIFIED 状态
   - 用户联系方式（姓名、届别、班级、邮箱）自动同步到校友名单（`WhitelistRoster`）
3. 或选择"撤销校友认证" → 用户降为 GUEST/PENDING

### 11.3 用户操作一览

所有操作通过 `POST /api/admin/users/[id]/actions` 统一入口，传入 `action` 参数：

| 操作 | action 值 | 效果 |
|------|-----------|------|
| 通过校友认证 | `approve-alumni` | 角色→ALUMNI，状态→VERIFIED，同步名册 |
| 撤销校友认证 | `reject-alumni` | 角色→GUEST，状态→REJECTED |
| 停用账号 | `disable-account` | accountStatus→DISABLED，所有会话立即失效 |
| 恢复账号 | `enable-account` | accountStatus→ACTIVE |
| 强制退出所有设备 | `logout-all-sessions` | sessionVersion+1，所有旧 token 失效 |
| 提升为管理员 | `grant-admin` | 授予 ADMIN 角色 |
| 撤销管理员 | `revoke-admin` | 撤销 ADMIN 角色，恢复为 ALUMNI 或 GUEST |
| 重发验证邮件 | `resend-verification` | 管理员代为触发邮箱验证邮件 |
| 发送密码重置邮件 | `send-reset-password` | 管理员代为触发密码重置邮件 |

### 11.4 V2.0 Root Admin 安全防护（重要）

系统内置三层防护，确保管理员之间的操作安全：

**第一层：Root Admin 唯一标识**
- 邮箱 `yanchuaner@yanchuaner.cn` 被硬编码为唯一的超级管理员（Root Admin）
- 可通过环境变量 `ROOT_ADMIN_EMAIL` 配置

**第二层：同级越权防御**
- 普通管理员（非 Root）对**其他管理员**执行以下操作时，后端返回 **403 Forbidden**：
  - `disable-account`（停用账号）
  - `revoke-admin`（撤销管理员）
  - `reject-alumni`（撤销校友认证）
  - `approve-alumni`（通过校友认证）
- 前端 UI 同步响应：目标为管理员时，操作按钮显示为 disabled 状态，提示"无权操作其他管理员"

**第三层：自保护机制**
- 管理员无法停用或撤销自己的管理员权限（返回 400 错误）
- 管理员无法撤销自己的校友认证
- 在用户列表页，当前登录管理员所在行的操作按钮被替换为"当前账号"标签（不可交互）

**角色与认证隔离规则**：
- 对已是 ADMIN 角色的用户，撤销校友认证按钮会被强制禁用，提示"请先撤销该用户的管理员权限"
- 正确操作流程：Root Admin 先执行"撤销管理员"将用户降为普通校友，再根据需要撤销其校友认证

## 12. 审计日志（V2.0 新增）

路径：`/admin/users`（用户操作日志隐含在操作记录中）

V2.0 新增的 `AuditLog` 表记录所有敏感管理操作，提供完整的可追溯性。

**记录结构**：

| 字段 | 说明 |
|------|------|
| `action` | 操作类型（如 `story-review-published`、`approve-alumni`、`disable-account`、`grant-admin`） |
| `targetType` | 操作目标类型（`Story`、`User`） |
| `targetId` | 操作目标 ID |
| `adminId` | 执行操作的管理员 ID |
| `before` | 操作前状态快照（JSON 字符串） |
| `after` | 操作后状态快照（JSON 字符串） |
| `createdAt` | 操作时间 |

**被审计的操作**：
- 故事审核：`story-review-published` / `story-review-rejected`
- 校友认证：`approve-alumni` / `reject-alumni`
- 账号控制：`disable-account` / `enable-account` / `logout-all-sessions`
- 权限变更：`grant-admin` / `revoke-admin`

**索引优化**：`adminId`、`(targetType, targetId)`、`createdAt` 三个索引覆盖常见查询场景。

## 15. 燕中记忆管理

路径：`/admin/memories`

管理文化长廊展示的校园记忆展品。**上传图片自动裁切为 16:9（2752x1548）**，与卡片展示比例一致。

### 15.1 新增展品

1. 填写**标题**（必填）、**副标题**（可选）、**描述**（可选）
2. 选择**图标**（六选一）：校门 / 教学楼 / 图书馆 / 操场 / 林荫道 / 相机
3. 上传或手动填写**图片路径**（可选）
4. 填写**图片 alt 说明**（可选）

### 15.2 排序、图片命名与编辑

- 点击 ↑/↓ 箭头调整展示顺序
- 保存时图片自动重命名为 `{板块英文名}{排序号}.jpg`（如 `campus0.jpg`、`building1.jpg`）
- 上传图片存入 `UPLOAD_DIR` 指定目录；本地默认 `public/uploads/`，生产推荐 `/var/www/alumni-site/uploads/`，前台通过 `/uploads/` 路径访问
- 前台为实时渲染（`force-dynamic`），保存后刷新即生效

## 18. 每日操作建议

### 每天登录后先做什么

1. 查看 `/admin` 控制面板是否有异常统计
2. 检查**故事审核工作台**（`/admin/stories/pending`）有无待审核投稿
3. 检查内容审核（`/admin/posts`）有无待处理内容
4. 检查用户审核（`/admin/users`）有无待认证校友
5. 检查信息修改申请（`/admin/alumni-corrections`）
6. 查看活动报名是否有新增数据
7. 如有新闻或活动要更新，先保存草稿预览，确认无误后再发布

### 高风险操作前

- 确认当前有可用备份（参考 [deployment.md](deployment.md) 中的备份章节）
- 批量导入前先在测试环境或小范围验证 CSV 格式
- 删除操作前确认数据是否已被引用

### 什么时候不要直接删除

- 活动已有报名记录时（系统会拒绝删除）
- 校友名单已被多处引用时
- 不确定数据是否已同步到前台时
- 故事审核通过或驳回后不可撤销，审核前确认正文内容

## 19. 问题排查指南

| 问题类型 | 排查方向 |
|----------|----------|
| 登录异常 | 检查账号状态、cookie 是否过期、`SESSION_SECRET` 是否正确、浏览器控制台错误 |
| 数据不一致 | 检查数据库和最近备份，确认是否有并发修改 |
| 上传失败 | 检查上传目录权限和磁盘空间 |
| CSV 导入失败 | 检查编码（必须 UTF-8）、表头格式、文件是否超过 2MB、是否至少有一行数据 |
| 页面空白或报错 | 查看浏览器控制台（F12），参考 [TROUBLESHOOTING.md](TROUBLESHOOTING.md) |
| 审核操作被拒绝(403) | 确认是否为 Root Admin；普通管理员不能操作其他管理员账号 |
| 投稿在前台不可见 | 确认故事状态是否为 PUBLISHED；检查 `/admin/stories/pending` 中是否有遗漏 |

## 20. 误操作处理

### 如果不小心删除了重要数据

1. **立即停止操作**，不要再修改其他内容
2. **备份当前数据库**（防止二次损坏）
3. 从最近的备份恢复数据
4. 确认数据完整性后再恢复服务

### 原则

- 没有备份的情况下不执行高风险操作（批量删除、导入覆盖等）
- 有真实报名数据的活动不删除
- 审核操作（故事通过/驳回、校友认证）不可撤销，操作前务必确认
- 删除操作不可撤销，操作前务必确认
