## 一、当前优先问题

1. 首页仍把技术、内容、运维和通讯参与者统一称为“平台共建者”，与实际分工不符。
2. 页脚和首页声明仍使用“个人公益版、个人发起”，没有体现两位技术共建者与燕中校友汇的责任关系。
3. 在校生资源站曾将 `/students/senior-qa` 卡片标题改成专题式标题“理工逐梦：以代码与公式丈量未来”，但落地内容仍是通用“学长问答”，标题与目标页面不匹配。
4. “校友寄语”等页面包含编辑整理的通用内容，不应表述为真实校友共同心声。真实投稿应明确署名方式、届别、来源和审核状态。
5. 学校介绍存在“重点建设 of 现代化”的错字；宣传性事实应基于可核实公开来源，无法确认时使用审慎表述。
6. 新闻、活动、联系我们、教师频道和在校生资源均要求认证后访问，公开访客难以判断平台价值、近期活跃度和联系渠道。后续应重新划分公开内容与校友隐私内容。
7. 多个错误页仍显示 `Signal Lost`、`Orbit Decayed`，需统一为中文。
8. “燕中故事”“燕川故事 · 轻论坛专栏”等栏目命名不统一。
9. 首页同时承载多层动画、公告、频道、动态、地图、成员和加入入口，内容过密，需要收敛主线。

## 二、统一命名与责任层级

建议固定为：

```text
产品名称：燕中校友数字母港
技术共建：詹勇弟、黄湘林、王俊（均为 2025 届）
运营维护：燕中校友汇
内容栏目：燕中故事、燕中记忆、在校生资源站等
```

“技术共建”用于说明网站产品、设计、开发和技术运维责任；“燕中校友汇”用于说明日常运营、内容维护、社群联络和资料整理。普通成员不统一称为技术共建者。

## 三、首页人员展示方案

首页人员区域分为上下两层。

### 1. 技术共建者

- 位于上层，固定展示三人，不做横向滚动。
- 当前名单和首页短文案如下，三人均为 2025 届：

| 姓名 | 首页职责文案 |
| --- | --- |
| 詹勇弟 | 协助开发与资源支持 |
| 黄湘林 | 核心设计与开发运维 |
| 王俊 | 功能完善与问题反馈 |

- 每人只展示姓名、届别、身份“技术共建者”和上述职责说明。
- 可选展示学校或城市，但不是必填项，不展示联系方式。
- 三张信息块在桌面端并排，移动端上下排列。
- 视觉权重高于运营成员，但不使用夸张个人宣传文案。

### 2. 燕中校友汇运营团队

- 位于下层，标题使用“燕中校友汇运营团队”或“运营与内容维护”，不再使用“全部共建者”。
- 按届别分成“2025 届”和“2026 届”两条单行横向滚动列表，避免 15 位成员挤在同一条轨道中。
- 每项固定宽度，展示姓名、当前分工和可选的学校/城市信息。
- 手机使用原生触摸滚动；桌面提供左右箭头按钮，并支持鼠标滚轮或触控板横向浏览。
- 使用 CSS `scroll-snap` 保持停靠整齐，不自动轮播，避免干扰阅读和无障碍体验。
- 首屏露出下一项的一部分，让用户明确列表可以横向滚动。
- 成员顺序采用明确规则，例如负责人优先，其余按姓名或加入时间排序，避免每次刷新随机变化。

### 3. 展示边界

- 只有本人同意公开的信息才能上线。
- 不展示手机号、私人邮箱、微信号、详细住址等个人信息。
- 不默认使用头像；缺少统一、授权的照片时，使用姓名首字或一致的图标占位。
- 离开运营团队后应及时移除或转入“历史贡献致谢”，避免名单长期失真。
- 如需保留其他临时贡献者，单独设置简洁的“特别感谢”，不要与当前运营团队混排。
